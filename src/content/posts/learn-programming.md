## ভূমিকা

প্রোগ্রামিং — এই শব্দটা শুনলেই অনেকে মনে ভয় পায়  । কিন্তু আসলে প্রোগ্রামিং শেখা ততটা কঠিন নয় যতটা আমরা ভাবি। এই পোস্টে আমি আমার নিজের অভিজ্ঞতা থেকে কিছু কথা বলব — কীভাবে শুরু করবেন, কোথায় শুরু করবেন, এবং কী কী ভুল এড়িয়ে চলবেন।

---

## কেন প্রোগ্রামিং শিখবেন?

প্রোগ্রামিং শুধু চাকরি পাওয়ার জন্য নয়। এটা একটা **চিন্তার পদ্ধতি** যা আপনার সমস্যা সমাধানের ক্ষমতা বাড়ায়।

- **সৃজনশীলতা:** নিজের আইডিয়াকে বাস্তবে রূপ দিতে পারবেন
- **স্বয়ংক্রিয়তা:** বারবার করা কাজ অটোমেট করতে পারবেন
- **ক্যারিয়ার:** বিশ্বজুড়ে কাজের সুযোগ তৈরি হবে
- **সমস্যা সমাধান:** লজিক্যাল থিংকিং উন্নত হবে

---

## কোন ভাষা দিয়ে শুরু করবেন?

নতুনদের জন্য আমার পরামর্শ হলো **Python** আর **C** পাশাপাশি শেখা। Python দিয়ে শুরু করলে দ্রুত কিছু বানিয়ে আত্মবিশ্বাস বাড়ে, আর C শেখালে প্রোগ্রামিং আসলে কম্পিউটারের ভেতরে কীভাবে কাজ করে এবং কঠিন প্রোগ্রামিং বলতে কী বোঝায়, সেটা পরিষ্কার হয়।

**Python** দিয়ে শুরু করার কারণ:

1. সিনট্যাক্স সহজ ও পরিষ্কার
2. বিশাল কমিউনিটি সাপোর্ট
3. ওয়েব ডেভেলপমেন্ট থেকে মেশিন লার্নিং — সব জায়গায় ব্যবহার হয়

**C** শেখার কারণ:

1. মেমরি, পয়েন্টার, ডেটা স্ট্রাকচার — এসবের ভিতরের ধারণা স্পষ্ট হয়
2. কম্পিউটার কীভাবে নির্দেশনা চালায়, সেটা কাছ থেকে দেখা যায়
3. কঠিন সমস্যার সামনে ধৈর্য, ডিবাগিং, আর প্রোগ্রামিংয়ের শৃঙ্খলা তৈরি হয়

python এ একটা সহজ উদাহরণ দেখুন:

```python
# আপনার প্রথম Python প্রোগ্রাম
name = input("আপনার নাম কী? ")
print(f"স্বাগতম, {name}! প্রোগ্রামিং জগতে আপনাকে অভিনন্দন।")
```

C-তেও একই ধারণা একটু নিচু স্তরে দেখতে পাবেন:

```c
#include <stdio.h>

int main(void) {
    char name[50];

    printf("Apnar nam ki? ");
    scanf("%49s", name);

    printf("Swagotom, %s! Programming jogote apnake obhinondon.\n", name);
    return 0;
}
```

## শেখার রোডম্যাপ

আমি একটা সহজ রোডম্যাপ প্রস্তাব করছি:

| সপ্তাহ | বিষয় | লক্ষ্য |
|--------|-------|--------|
| ১-২ | ভেরিয়েবল, ডেটা টাইপ, কন্ডিশন | বেসিক লজিক বোঝা |
| ৩-৪ | লুপ, ফাংশন | কোড পুনর্ব্যবহার শেখা |
| ৫-৬ | লিস্ট, ডিকশনারি | ডেটা স্ট্রাকচার |
| ৭-৮ | ফাইল হ্যান্ডলিং, এরর হ্যান্ডলিং | বাস্তব সমস্যা সমাধান |
| ৯-১২ | একটা ছোট প্রজেক্ট তৈরি | সব কিছু একত্রে প্রয়োগ |

## সাধারণ ভুলগুলো এড়িয়ে চলুন

### ১. শুধু টিউটোরিয়াল দেখে যাওয়া

অনেকে ঘণ্টার পর ঘণ্টা টিউটোরিয়াল দেখেন কিন্তু নিজে কোড লেখেন না। এটা "tutorial hell" নামে পরিচিত। প্রতিটা নতুন কনসেপ্ট শেখার পর নিজে কিছু লেখার চেষ্টা করুন।

### ২. প্রথমেই জটিল প্রজেক্ট শুরু করা

ধৈর্য ধরুন। ছোট ছোট প্রোগ্রাম দিয়ে শুরু করুন — ক্যালকুলেটর, টু-ডু লিস্ট, সংখ্যা অনুমান খেলা ইত্যাদি।

### ৩. একসাথে অনেক ভাষা শেখার চেষ্টা

একটা ভাষায় ভালো দক্ষতা অর্জন করুন, তারপর অন্য ভাষায় যান।

## একটা ছোট প্রজেক্ট: সংখ্যা অনুমান খেলা

নিচে একটা সহজ গেম দেওয়া হলো যেটা আপনি শেখার প্রথম সপ্তাহেই বানাতে পারবেন:

```python
import random

গোপন_সংখ্যা = random.randint(1, 100)
চেষ্টা = 0

print("আমি ১ থেকে ১০০ এর মধ্যে একটা সংখ্যা ভেবেছি। অনুমান করুন!")

while True:
    অনুমান = int(input("আপনার অনুমান: "))
    চেষ্টা += 1

    if অনুমান < গোপন_সংখ্যা:
        print("আরো বড় সংখ্যা!")
    elif অনুমান > গোপন_সংখ্যা:
        print("আরো ছোট সংখ্যা!")
    else:
        print(f"দারুণ! আপনি {চেষ্টা} বার চেষ্টায় সঠিক উত্তর পেয়েছেন!")
        break
```
